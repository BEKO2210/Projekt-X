# specs/

## Zweck

Detaillierte **Funktionsspezifikationen** pro Modul, Feature oder Workflow. Ergänzen, aber ersetzen nicht: `MVP_SPEC.md`, `ARCHITECTURE.md`, `DATA_MODEL.md`.

## Konvention (geplant)

- Dateiname: `<bereich>/<feature>.md`.
- Pflichtteile: Zweck, Nutzerwert, Scope, Nicht-Ziele, Datenschutz, Sicherheit, Quellenbedarf, Testfälle, Abhängigkeiten.

## Stand Run 1

Leer. Erste Specs entstehen ab Run 2 mit Vertiefung der MVP-Grenze.
