# releases/

## Zweck

**Release-Dokumentation**: jeder Übergang Dev → Test bzw. Test → Prod wird mit Datum, Inhalt, Gate-Ergebnis und Rückrollplan festgehalten.

## Konvention (geplant)

- Dateiname: `YYYY-MM-DD-<release-name>.md` oder `release-vX.Y.Z.md`.
- Pflichtteile: Änderungen, Gate-Ergebnisse (Dev-to-Test, Test-to-Production), Tests, Datenschutzprüfung, Sicherheitsprüfung, Rollback.
