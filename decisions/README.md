# decisions/

## Zweck

**Architecture Decision Records (ADR)**: jede wichtige technische oder strukturelle Entscheidung wird als einzelne Datei dokumentiert.

## Konvention (geplant)

- Dateiname: `YYYY-MM-DD-<kurztitel>.md` oder `ADR-NNN-<kurztitel>.md`.
- Inhalt: Kontext, Entscheidung, Alternativen, Konsequenzen, Status (`vorgeschlagen`/`akzeptiert`/`ersetzt`).

## Stand Run 1

Leer. Erste ADR wird erwartet spätestens zur Techstack-Entscheidung (vor Phase 1-Implementierung).
