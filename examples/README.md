# examples/

## Zweck

**Beispielabläufe und Beispielausgaben** zur Veranschaulichung von Workflows, Dokumenten und Agenten-Outputs.

## Harte Regeln

- **Nur synthetische Daten.** Keine realen Personen, Firmen, Adressen, Nummern, Behördenbezüge.
- **Keine Secrets.**
- **Keine Außenwirkung.** Beispiele illustrieren Struktur, nicht echten Versand.
- Jedes Beispiel ist im Kopf als „Beispiel / illustrativ" gekennzeichnet.

## Stand Run 1

Leer. Beispiele entstehen begleitend zu den späteren Workflow- und Dokumenttypen.
